# AI Rules - TT Daily Task (Optimized for React 19 & TypeScript)

## 🛠 Tech Stack
- **Framework:** React 19 (Strict) | **Vite 8** | **TypeScript** (Strict Mode)
- **Styles:** Tailwind v4 | **Icons:** Lucide | **Animations:** Framer Motion
- **Backend:** Firebase (Firestore, Auth, Storage)

## 🏗 Architecture & Discipline
- **Hybrid TS/JS:** Favor `.ts/.tsx` for new files. Refer to `src/utils/helpers.ts` for interfaces.
- **Hooks-First:** Business logic must be in `src/hooks/`. Components only orchestrate.
- **Service Layer:** API calls must be in `src/services/`.
- **File Length:** STRICT limit of 300 lines. Refactor if exceeded.
- **State:** Use Zustand for global state. Avoid prop drilling > 5 props.

## 🚀 React 19 & Compiler Guardrails (CRITICAL)
- **NO Synchronous setState in useEffect:** Use `requestAnimationFrame(() => set...)` to avoid cascading renders.
- **Immutability:** Do not mutate properties of objects from Refs/Hooks directly (e.g., Audio). Use `Object.assign` or cloning.
- **Audio:** Maintain the `globalAudio` singleton pattern outside components to prevent re-initialization.

## 🧹 Quality Control
- **Zero Lint Errors:** Run `npm run lint` before finishing.
- **Unused Vars:** REMOVE all unused variables/imports. (Regex ignore: `^[A-Z_]`).
- **Context:** Run `npm run pack` (repomix) to update context for other AI sessions.

## 📝 Communication & Guidelines
- **Ngôn ngữ:** Tiếng Việt.
- **Golden Rule:** ALWAYS use TypeScript (.ts/.tsx) for new logic or components.
- **Master Guidelines:** Refer to `GUIDELINES.md` for detailed dev standards.
- **Scope:** ONLY modify files related to the specific task. Preserving codebase integrity is priority.

## 🎨 Design Aesthetics
- **Theme:** Dark Mode, Glassmorphism, Premium feel, Micro-animations.

## 🔑 Key Concepts

### 1. **Architecture Pattern: React + Firebase + Zustand**
- **State Management**: Zustand (useAppStore) for global state
- **Firebase**: Real-time sync for tasks, users, daily quests
- **Contexts**: TaskActionContext for cross-component task actions
- **Hooks**: Custom hooks for business logic isolation

### 2. **Main Entry Hook: useTTApp()**
```typescript
// src/hooks/useTTApp.ts - THE BRAIN OF THE APP
const {
  user,                      // Firebase auth user
  userData,                  // User profile + stats
  tasks,                     // All tasks in app
  dailyQuest,               // Daily quest object
  taskActions,              // { toggleTaskStatus, handleUpdateTask, ... }
  config,                   // Calendar API keys
  focusingTaskId,           // Currently focused task
  // ... 20+ more properties
} = useTTApp();
```

### 3. **Heartbeat System (useHeartbeat.ts)**
- Sends heartbeat every 90s when task is running
- Marks tasks as `autoPaused` if no heartbeat for 6 mins (stale)
- Boot cleanup: Auto-pause any stale tasks on app load

### 4. **Calendar Auto-Sync (useCalendarAutoSync.ts)**
- Fetches Google Calendar events every 30 mins
- Creates automated tasks from calendar events
- Skips heartbeat for automated tasks (`isAutomated: true`)

### 5. **Auto Task Logic (useAutoTaskLogic.ts)**
- Auto-starts tasks at `scheduledStartTime`
- Auto-completes tasks at `scheduledEndTime`
- Only for tasks with schedule info from calendar

## 🎯 Common Patterns

### 1. **Task Action Pattern** (useTaskActions.js)
```typescript
taskActions.toggleTaskStatus(id, 'start' | 'pause' | 'complete');
taskActions.handleUpdateTask(id, updates);
taskActions.handlePriorityChange(id, priority?);
```

### 2. **Firebase Subscription Pattern**
```typescript
const unsubscribe = userService.subscribeToUserStats(uid, (data) => {
  // real-time updates
});
return () => unsubscribe(); // cleanup
```

### 3. **Effect with Auto-cleanup**
```typescript
useEffect(() => {
  const interval = setInterval(syncData, 30000);
  return () => clearInterval(interval);
}, [dependencies]);
```
